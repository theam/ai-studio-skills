# Frameworks ligeros para ingenieros (sin humo)

## Core loop: Build → Measure → Learn
- **Build**: entrega el mínimo que produce señal.
- **Measure**: define eventos, métricas, cohorts.
- **Learn**: decide: iterar, ampliar, pivotar o parar.

## MVP (Minimum Viable Product)
Pregunta guía: “¿Cuál es la versión más pequeña que valida o invalida la hipótesis?”

## RICE / ICE (priorización rápida)
- **RICE**: Reach, Impact, Confidence, Effort
- **ICE**: Impact, Confidence, Ease

Úsalos como conversación, no como religión.

## Jobs To Be Done (JTBD)
“Cuando estoy en situación S, quiero lograr progreso P, para obtener resultado R.”

Ayuda a no construir features sueltas.

## Opportunity Solution Tree (Teresa Torres)
- Oportunidad (dolor) → señales → soluciones → experimentos
Buen formato para coordinar equipo y evitar “lista de features”.

## PRD / One-pager (mínimo)
- Problema
- Usuario
- Contexto
- Propuesta
- Métricas
- Alcance/No-alcance
- Riesgos

Usa `assets/templates/product-brief.md`.

## Métricas: North Star y leading indicators
- North Star: una métrica que captura valor continuo.
- Leading indicators: señales tempranas que predicen la North Star.

## “Definition of Done” orientado a producto
No es “mergeado”: es “entregado + medible + adoptado”.
